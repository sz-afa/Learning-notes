# gcc
## 文件类型:
| 后缀名 | 类型 |  
| :----: | :---- |
| .c | c源代码 | 
| .h | 程序包含的头文件 | 
| .i | 预处理后的c源代码 |
| .s | 汇编源代码 |
| .o | 编译后文件 |
## 流程:
### 1.预处理:
命令
```bash
gcc -E test.c -o test.i
```
展开头文件。替换宏。删除注释。条件编译。
### 2.编译
命令
```
gcc -S test.i -o test.s
```
-S让编译器在编译之后停止，不进行后续过程
### 3.汇编
```
gcc -c test.s -o test.o
```
将汇编代码转换成二进制机器码
### 4.链接
```
gcc test.o -o test
```
# gdb
## 简介:
gdb调试: 一般用于查看段错误
被调试的c代码编译时需要加-g参数，否则无法调试
```bash
gcc 1.c -g
gdb a.out
```
## 常用命令
|命令| 含义 |
|:----|:----|
|l|列出文件内容|
|b n |在第n行设置断点|
|info b|查看断点信息|
|r|运行程序|
|p i |查看i变量的值|
|c|继续运行，直到断点处停止|
|n/s|单步运行|
|del n|删除n号断点|
|q|退出|
